Evidence Checker

医学・健康情報のエビデンス自動評価システム

v0.3.0 for macOS

1セットアップ

必要なツールをインストールします：

# Homebrewでツールをインストール $ brew install poetry jq httpie # プロジェクトディレクトリに移動 $ cd "/Users/tkojima/claude code/Evidence_Checker" # 依存関係のインストール $ make install

2サーバー起動

Evidence Checkerを起動します：

# 開発サーバー起動 $ make run # または直接起動 $ poetry run uvicorn src.main:app --reload
起動後、http://localhost:8000 でアクセスできます

3API使用方法

cURLを使用

# 基本的な評価リクエスト $ curl -X POST "http://localhost:8000/api/v1/score" \ -H "Content-Type: application/json" \ -d '{ "claim_text": "ビタミンDは免疫機能をサポートする", "topic": "health", "lang": "ja" }' | jq '.' # より詳細な情報を含むリクエスト $ curl -X POST "http://localhost:8000/api/v1/score" \ -H "Content-Type: application/json" \ -d '{ "claim_text": "定期的な運動は心臓病のリスクを30%減少させる", "source_url": "https://example.com/health-article", "topic": "health" }' | jq '.total_score, .label, .rationales'

HTTPie（推奨）

# シンプルで読みやすい構文 $ http POST localhost:8000/api/v1/score \ claim_text="ビタミンDは免疫機能をサポートする" \ topic=health \ lang=ja # 有害な情報のテスト $ http POST localhost:8000/api/v1/score \ claim_text="ワクチンは危険で不要です" \ topic=health

ブラウザでテスト

# Swagger UIを開く $ open http://localhost:8000/docs # ヘルスチェック $ open http://localhost:8000/health/

Swagger UIでは、インタラクティブにAPIをテストできます。

4Docker使用

Docker環境での実行：

# Docker Composeで起動 $ make docker-compose-up # ログの確認 $ docker-compose logs -f evidence-checker # 停止 $ make docker-compose-down

5テスト実行

システムの動作確認：

# 全テスト実行 $ make test-all # APIテストのみ $ make test-api # パフォーマンステスト $ make benchmark

6実用例

よくある使用パターン：

📝 ファイルから主張を読み込み

# テスト用ファイルの作成 $ echo "ビタミンCは風邪予防に効果的です" > claim.txt # ファイルから読み込んで評価 $ http POST localhost:8000/api/v1/score \ claim_text="$(cat claim.txt)" \ topic=health

🔄 複数の主張を連続評価

# スクリプトファイルの作成 $ cat > test_claims.sh << 'EOF' #!/bin/bash claims=( "ビタミンDは免疫機能をサポートする" "定期的な運動は心臓病リスクを減少させる" "ワクチンは危険で不要です" ) for claim in "${claims[@]}"; do echo "=== 評価中: $claim ===" http POST localhost:8000/api/v1/score \ claim_text="$claim" \ topic=health | jq '.total_score, .label' echo "" done EOF # 実行権限を付与して実行 $ chmod +x test_claims.sh && ./test_claims.sh

🛠️ 便利なエイリアス

~/.zshrc に追加すると便利：

# Evidence Checker用エイリアス alias ec-start="cd '/Users/tkojima/claude code/Evidence_Checker' && make run" alias ec-test="cd '/Users/tkojima/claude code/Evidence_Checker' && make test-all" alias ec-check="http GET localhost:8000/health/" alias ec-eval="http POST localhost:8000/api/v1/score claim_text=" # 使用例 $ ec-eval "ビタミンDは健康に良い"
